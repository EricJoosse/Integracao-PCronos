Checklist upgrade versão Manual TI no Eclipse
=============================================

- PRIMEIRO, ANTES DE EDITAR A ALTERAÇÃO NO MANNUAL:
  No caso que a versão atual do .jar no Eclipse project já está em produção em algum servidor de um fornecedor:
  aumentar a versão do .jar também, veja "Checklist upgrade versão .jar no Eclipse.txt"

- Editar o conteúdo do manual em ManualManutencao.java
	- aumentar a versão no título (no método setNomeArquivoMaisAtalho())
	- as alterações necessárias   (no constructor da clasase)

- Editar a geração do nome do arquivo do manual (a versão) em ManualManutencao.java (em 3 lugares)

- Editar o nome do arquivo Info Geral (a versão) no projeto Eclipse

- Editar o atalho no menu de Windows "Info Geral" (a versão) em ManualManutencao.java 

- Compilar o novo .jar

- Atualizar o arquivo "Tabela Versão .jar X Versão Manual TI.txt"

- Re-gerar o Manual TI no raiz do projeto Eclipse para acompanhamento dos setores TI dos forneceores
  (usando GeradorNaHoraManualTI_COM_PARAMETRO_IdFornecedor.launch e passando o idFornecedor como parâmetro - default = 947).
    - Testar o manual gerado
    
    - Mover a versão anterior para: 
      C:\PCronos\Integração Fornecedor - Portal Cronos\Instalador\Windows\Arquivos de Programas PC\Integração Fornecedor - Portal Cronos - Versões anteriores\
       
  	- Talvez apagar versões anteriores no raiz do projeto Eclipse apenas se não estão 
	  usadas mais em nenhum servidor de nenhum fornecedor


- Criar ou editar novo "Manual Upgrade Ambientes de Produção de x.y(.z) para a.b(.c).txt"

5.	(i) Editar o conteúdo do Manual TI:
		- aumentar a versão no título
		- Substituir:

  - .......

          Por:
  - .......
		
	(ii)  Renomear o nome do arquivo do Manual TI (aumentar a versão de x.y.z para x.y.z e a data de dd.mm.yyyy para dd.mm.yyyy)
	(iii) Editar o atalho no menu de Windows para apontar para o arquivo renomeado
	(iv)  Se precisar (depende da versão de Windows), renomear o atalho 

- Se precisar, atualizar o instalador

- Agendar atualização de todos os servidores dos fornecedores
