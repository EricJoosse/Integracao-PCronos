Checklist upgrade versão .jar no Eclipse
========================================

- Sempre usar o formato integr-fornecedor-x.y.z.jar e nunca usar o formato integr-fornecedor-x.y.jar,  
  até no caso 7.0.0 por exemplo, por motivos de ordenação de diversos tipos de arquivos e alinhamentos.
  
- Editar pom.xml (4 lugares)
	- Cuidado para não alterar versões de dependências de outros jars por engano

- Project > Maven > Update Project

- Editar bin\Versao.bat no Instalador

- Compilar o .jar

- Mover o .jar anterior para 
  \Instalador\Windows\Arquivos de Programas PC\Integração Fornecedor - Portal Cronos - Versões anteriores\

- Registrar uma pendência nova para criar o próximo tag novo no futuro e colocar esta pendência 
  na posição correta da fila de commits, no lugar mais adiantado possível, no último momento possível. 
  Já registrar logo agora a descrição do tag futuro. 

- Se precisar, criar um novo arquivo "Manual Upgrade Ambientes de Produção de x.y.z para a.b.c.txt" 
  copiando o arquivo "Manual Upgrade Ambientes de Produção de OUTRA para OUTRA.txt"
 
- Atualizar o arquivo "Tabela Versão .jar X Versão Manual TI.txt" repetindo a última linha 
  se não houve alteração no Manual TI
 
- Dentro do arquivo "Passo a passo geração ou re-geração do instalador Setup.exe.txt",  
  por exemplo item 3, no campo "Nome":
  atualizar a versão dentro do nome do arquivo "Instalador do Integrador Fornecedores - Portal Cronos.*.exe" 
  em 4 lugares. 
 
- Se precisar: atualizar o instalador, ou registrar uma pendência. 
  Geralmente não precisa, pois a atualização do instalador já é um dos passos na instalação do Integrador 
  em um servidor novo, (enquanto que ainda não existe uma tabela nova dbo.Configuracao_Integrador). 

- Commitar no GitHub:
	- o instalador da versão anterior do Integrador,  sem JRE! (para minimizar o tamanho no GitHub)
	- o instalador da versão anterior do Monitorador, sem JRE! (para minimizar o tamanho no GitHub)
	- a remoção da versão antes disso do instalador do Integrador
	- a remoção da versão antes disso do instalador do Monitorador
